Functional Building Blocks (WIP)

In an ideal system our workflow would be made up of a series of composed computations. Those computations would be represented by small building blocks that could be reused through out the whole system. For example, computations that would perform operations like:

  • adding numbers
  • manipulating strings
  • iterating over data

would now be modeled as bite-size, composable pieces.

add       = (a, b) -> a + b
first     =    (a) -> a[0]
lowercase =    (a) -> a.toLowerCase()

Above we've modeled several of our building blocks as ordinary functions. Now we can begin to compose our computations by making sure our functions adhere to a pattern. All functions take in their value(s) and return a value.

When using this pattern we're able to ensure that our blocks will compose and collaborate properly. e.g.

shrinkFirst =    (a) -> lowercase (first a)
abbreviate  = (a, b) -> add (shrinkFirst a), (shrinkFirst b)

The same methodology can be applied over and over again as we build up to our desired results.
